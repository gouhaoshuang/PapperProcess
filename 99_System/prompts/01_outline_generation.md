# 大纲生成提示词 (Outline Generation Prompt)

## 使用说明

- **阶段**: 阶段一 - 大纲生成
- **输入**: PDF 论文文件 (或转换后的 Markdown)
- **输出**: JSON 格式的结构化大纲（包含详细子要点）

---

## 提示词模板

我现在给你上传了一份 PDF 文件（论文），请进行全文理解。

### 任务

请生成一份 **详细的笔记大纲**，大纲采用 **JSON 格式** 返回，便于后续程序解析和逐段落扩写。

### JSON 格式要求

```json
{
  "paper_title": "论文标题（中英文皆可）",
  "authors": "作者信息（如有）",
  "year": "发表年份（如有）",
  "one_liner": "用一句话总结整篇论文的核心贡献（50-80字）",
  "sections": [
    {
      "id": 1,
      "title": "第一部分标题 (例如: 背景与动机)",
      "subsections": [
        {
          "subtitle": "子标题 (例如: 移动端部署LLM的挑战)",
          "key_points": [
            "需要解释的关键概念或论点1",
            "需要解释的关键概念或论点2",
            "需要解释的关键概念或论点3"
          ]
        },
        {
          "subtitle": "子标题 (例如: 激活稀疏性原理)",
          "key_points": [
            "需要解释的关键概念或论点1",
            "需要解释的关键概念或论点2"
          ]
        }
      ],
      "key_figures": ["图片 1", "图片 3"],
      "key_formulas": true
    },
    {
      "id": 2,
      "title": "第二部分标题 (例如: 方法论 - 离线优化阶段)",
      "subsections": [
        {
          "subtitle": "子标题 (例如: 模式提取方法)",
          "key_points": ["共激活概率的计算方式", "距离度量的定义"]
        },
        {
          "subtitle": "子标题 (例如: 图抽象与哈密顿路径)",
          "key_points": ["如何将问题建模为图论问题", "贪心算法的设计思路"]
        }
      ],
      "key_figures": ["图片 2"],
      "key_formulas": false
    }
  ]
}
```

### 内容规则

2. **`title`**：每个部分的主标题，应清晰反映该部分的核心主题（如"离线阶段：相关性感知聚类"）。
3. **`subsections`**：每个部分包含 **2-5 个子标题**，用于细化该部分的内容结构。
4. **`key_points`**：每个子标题下列出 **2-5 个关键要点**，这些要点应该是：
   - 需要在后续笔记中详细解释的**术语或概念**（如"IOPS瓶颈"、"神经元共激活"）
   - 需要回答的**问题**（如"为什么传统布局不适合稀疏推理？"）
   - 需要阐述的**核心论点**（如"牺牲带宽换取IOPS降低的策略"）
5. **`key_figures`**：列出该部分涉及的图片编号（使用 `图片 1`、`图片 2` 等格式）。
6. **`key_formulas`**：该部分是否包含重要的数学公式。
7. **完整性**：不要遗漏实验结果分析、消融实验、敏感性分析等重要章节。

### 输出要求

**请直接输出 JSON，不要添加任何 Markdown 代码块标记或其他说明文字。**
