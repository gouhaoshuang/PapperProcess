# 段落扩写提示词 (Section Expansion Prompt)

## 使用说明

- **阶段**: 阶段二 - 逐段扩写
- **输入**: 段落标题、子标题与关键要点列表、相关图片列表
- **输出**: Markdown 格式的详细笔记内容

---

## 提示词模板

### 背景

你已阅读了完整的论文内容（已作为上下文/缓存提供）。

### 当前任务

请根据以下大纲段落，生成 **详细的笔记内容**：

**段落标题**: {section_title}

**子标题与关键要点**:
{subsections_text}

**相关图片**: {key_figures}
**包含公式**: {key_formulas}

### 生成规则

1. **按要点逐一展开**: 对每个 `key_point` 进行详细解释：
   - 解释清楚每个术语的含义
   - 回答每个提出的问题
   - 阐述每个核心论点的逻辑
2. **数学公式**: 设计到的每一个数学公式，请按照 latex 的格式给出，公式的前后需要各添加一 $ 。 示例：乘法交换律公式：$a \cdot b = b \cdot a$。
3. **排版美观**: 在英文单词和数字的前后各加一个空格，保持美观。
4. **图片分析**: 对于相关图片中列出的图片，请进行精讲分析。使用 `<图片 X>` 格式引用（不需要给出实际图片）。
5. **内容深度**:
   - 解释核心概念和术语
   - 阐述方法论细节
   - 分析实验数据和结果
   - 总结关键结论
6. **结构清晰**: 使用适当的 Markdown 标题层级组织内容。

### 输出格式

直接输出 Markdown 格式的笔记内容。使用以下结构：

- 使用 `###` 作为子标题（对应大纲中的 `subtitle`）
- 使用 `####` 作为更细粒度的小节标题
- 使用列表、表格等元素增强可读性

---

## 变量说明

| 变量名               | 类型   | 说明                                           |
| -------------------- | ------ | ---------------------------------------------- |
| `{section_title}`    | string | 段落主标题，来自大纲 JSON                      |
| `{subsections_text}` | string | 子标题与关键要点的格式化文本（见下方格式示例） |
| `{key_figures}`      | string | 相关图片列表，逗号分隔                         |
| `{key_formulas}`     | string | 是否包含重要公式（"是" 或 "否"）               |

### `{subsections_text}` 格式示例

```
- **移动端部署LLM的挑战**
  - 内存（DRAM）限制 vs. 模型参数量巨大
  - 隐私与个性化需求推动端侧部署
- **激活稀疏性原理**
  - ReLU等激活函数导致大量神经元输出为0
  - 现有方案：将模型存放在Flash中，仅将激活的神经元加载到DRAM
```
